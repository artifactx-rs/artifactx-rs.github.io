The heavy feature

Migrate from an existing repo without rebuilding every package.

The import code downloads package files from upstream repository metadata into the local ArtifactX pool. That means you can mirror a bounded slice first, publish it, point test clients at ArtifactX, then widen the import when you are ready.

apt import

Package metadata first, files second.

For apt, ArtifactX tries Packages.gz, falls back to plain Packages, parses Package and Filename, then downloads matching files into the configured component pool.

yum import

Repodata-driven RPM migration.

For yum/dnf, ArtifactX reads repodata/repomd.xml, finds primary metadata, extracts package href entries, then downloads RPMs into the configured repo directory.

safe rollout

Filter before you switch traffic.

Use --limit for small trial imports. On apt, add --arch and --match-name to migrate only the architecture or package prefix you need first.

same engine

CLI and server API share the path.

The running server exposes POST /api/v1/import, using the same apt/yum import logic behind arx import for remote migration workflows.

The product in four moves

A repo pipeline you can read at a glance.

No visual logo wall, no platform maze. ArtifactX is a small operational loop: import or add packages, publish signed metadata, then serve or push.

01

Initialize

arx init creates the repository layout, config, and signing key.

02

Import or add

arx import migrates from upstream apt/yum metadata; arx add accepts local .deb/.rpm files.

03

Publish

arx publish signs and indexes apt/yum metadata with atomic staging.

04

Operate

serve, push, rollback, gc, and history keep the repo boring.

Why it exists

Move off a repo manager without a flag day.

Import a small slice from the current apt/yum source, publish your signed copy, test clients against ArtifactX, then expand the migration when the path is proven.

Import

Existing repos become your pool.

arx import downloads packages listed by upstream apt/yum metadata and skips files that already exist locally.

Sign

PGP metadata by default.

Generated apt/yum metadata is signed so standard Linux package managers can verify it.

Recover

Rollback is a first-class command.

arx history lists retained states. arx rollback flips back to a previous published state instead of rebuilding a rescue plan under pressure.

Package

Still build new packages.

arx pack builds .deb, .rpm, and optional .apk output from a manifest or Cargo.toml.

Automate

Push to a running server.

arx push uploads packages to arx serve using token auth or GitHub Actions OIDC.

Deploy

Compose files are generated.

arx compose is a real CLI command that writes Docker deployment files for a repo root and listen address.

verified examples 
# Bounded apt migration
arx import <apt-repo-url> --apt --dist stable --component main --arch amd64 --match-name myapp --limit 20 --root ./repo

# Bounded yum migration
arx import <yum-repo-url> --yum --component production --limit 20 --root ./repo

arx publish --root ./repo

# Routine maintenance
arx history --root ./repo
arx rollback stable --root ./repo
arx rm myapp --version 1.2.3 --root ./repo
arx gc --keep 3 --dry-run --root ./repo
